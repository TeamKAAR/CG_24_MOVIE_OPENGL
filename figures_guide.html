Figure 1 — System Architecture Diagram

How to Generate

The system architecture diagram is provided as a Mermaid flowchart source in system_architecture.mmd. Use one of the following tools to render it as a high-resolution PNG or SVG for the book chapter:

🌐 mermaid.live 🖥 draw.io → Extras → Edit Diagram ⚡ npx @mermaid-js/mermaid-cli

CLI Export (Recommended for Print Quality)

npm install -g @mermaid-js/mermaid-cli mmdc -i system_architecture.mmd -o fig1_architecture.png -w 2400 -H 1600 -b transparent

This produces a 2400×1600 px PNG suitable for print at 300 DPI. Use -b white if the publication requires a white background.

Architecture Overview

The diagram shows five layers: the GLUT Event / Time Controller layer at the top driving the scene clock; the Scene Dispatch layer routing to one of six draw functions; the Rendering Subsystems layer (Stickman, Rain, Gears, Bloom, Text, Texture); the Projection System (3D perspective + 2D ortho overlay); and the GPU Rasteriser (fixed-function OpenGL immediate mode) producing the final double-buffered frame.

Screenshot Capture Tips

Controls

SPACE — skip to next scene  ·  P — pause / unpause  ·  ESC — quit

Use P to pause at the right moment, then your OS screenshot shortcut:
Linux : gnome-screenshot --window OR scrot -u screenshot.png Windows: Win+Shift+S (Snip & Sketch) OR PrintScreen macOS : Cmd+Shift+4 → select window

Increasing Native Resolution (Optional for Print)

Edit the two constants at the top of 24_film.cpp:

static const int W = 1920, H = 1080; // or 2560 x 1440 for very high quality

Recompile and run. The program will open at the new resolution. All scene geometry scales automatically because it is defined in normalised world-space coordinates.
